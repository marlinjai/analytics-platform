# Analytics Platform — Q2 2026 Roadmap

## Current State Assessment (updated 2026-03-22)

**Grade: B+ → A- (functional MVP, deployed to production)**

**Completed before Q2:**
- Phase 0-5 complete (scaffold → production deployment → polish)
- Phase 6 partially complete (UTM, referrer, browser/OS/device breakdowns, sources)
- Production live at analytics.lumitra.co (Hetzner + Terraform + Caddy)
- Tracker SDK published (@marlinjai/analytics-tracker, <5KB)
- First client integration (Lola Stories)
- 69 passing tests, migration tooling, skeleton UI, onboarding flow
- Conventional commits enforced via commitlint + husky

**Still missing for competitive parity:**
- GeoIP (country/city data)
- Dashboard filters + click-to-filter
- Remote SDK configuration (toggle features from dashboard, not client code)
- Data export
- Funnel analysis
- Real-time visitors
- Mobile navigation

## Month 1: Foundation & Core Analytics (Weeks 1-4)

### Week 1: Dashboard UI Overhaul

**Goal:** Transform the bare-bones dashboard into a polished, single-page analytics experience.

- [ ] **Redesign Overview page** — Single-page layout inspired by Plausible/Fathom
  - Hero KPI cards row (Visitors, Pageviews, Bounce Rate, Avg Duration, Current Visitors)
  - Full-width area chart (not line chart) with gradient fill
  - Two-column grid below: Top Pages | Top Sources
  - Two-column grid: Countries | Devices (Browser/OS/Screen)
- [ ] **Add mobile navigation** — Hamburger menu + slide-out drawer for sidebar
  - Current sidebar is `max-md:hidden` — mobile users see nothing
- [ ] **Implement date range picker** — Replace basic presets with proper picker
  - Calendar-based date selection
  - Period comparison toggle (overlay previous period as dotted line)
  - Shareable URL state (encode date range in query params)
- [x] ~~**Add skeleton loading states**~~ — DONE (Phase 5)
- [ ] **Remote SDK configuration** — Toggle features from dashboard, not client code
  - New API: `GET /api/projects/{id}/config` (returns enabled features)
  - Tracker fetches config on init, dynamically enables replay/heatmap/scroll
  - Dashboard Settings: toggles for Replay, Heatmap, Scroll Tracking per project
  - Eliminates need for clients to update code when changing features
- [ ] **Click-to-filter** — Click any table row to filter entire dashboard
  - Click a page → filter all metrics to that page
  - Click a country → filter to that country
  - Breadcrumb trail showing active filters with clear button

### Week 2: Traffic Sources & Technology Breakdown — DONE (completed in Phase 6)

- [x] ~~**Referrer tracking**~~ — Sources table with favicons, grouped by domain
- [x] ~~**UTM parameter tracking**~~ — Parsed in tracker SDK, stored in event properties
- [x] ~~**Technology breakdown**~~ — Browser, OS, device type tables on dashboard

### Week 3: Geographic Data

**Goal:** Show where visitors come from.

- [ ] **GeoIP integration** — Replace stub in `enrich.ts`
  - Integrate MaxMind GeoLite2 database (free, updated monthly)
  - Or use ip-api.com/ipinfo.io for lightweight lookups
  - Extract: country, region, city
  - Update ClickHouse schema: add `region`, `city` columns
- [ ] **World map visualization** — Choropleth map component
  - Use lightweight map library (e.g., react-simple-maps or custom SVG)
  - Color intensity based on visitor count
  - Click country to filter dashboard
- [ ] **Countries/Regions table** — Sortable table with flag emojis
  - Show: Country, Visitors, Pageviews, Bounce Rate
  - Drill down: Country → Regions → Cities

### Week 4: Real-Time Analytics

**Goal:** Show live visitor activity.

- [ ] **Current visitors counter** — Live visitor count in header/KPI cards
  - Count unique sessions active in last 5 minutes
  - New API: `GET /api/stats/realtime`
  - WebSocket or polling (15-second intervals)
  - Animated count transitions
- [ ] **Live event feed** — Optional panel showing events as they happen
  - New page or dashboard widget
  - Last 50 events with type, URL, country, device
  - Auto-scroll with pause on hover
- [ ] **Auto-refresh dashboard** — Optional toggle to refresh stats every 30s
  - Visual indicator when data is refreshing
  - Don't lose scroll position or filter state

## Month 2: Advanced Features & Polish (Weeks 5-8)

### Week 5: Heatmap In-Dashboard & Enhanced Replay

**Goal:** Bring heatmap visualization into the dashboard. Improve replay UX.

- [ ] **In-dashboard heatmap viewer** — Iframe-based page preview with overlay
  - Load target page in sandboxed iframe
  - Overlay heatmap.js visualization on top
  - Controls: URL selector, device toggle, date range
  - Screenshot mode (capture static image of heatmap)
- [ ] **Scroll depth heatmap** — New visualization type
  - Show how far users scroll on each page
  - Color-coded vertical bar (green → yellow → red)
  - Average fold line indicator
- [ ] **Replay improvements**
  - Session search & filter (by URL, country, device, duration)
  - Session tags/notes (mark interesting sessions)
  - Thumbnail previews in session list
  - Skip to click events in timeline
  - Rage click detection (highlight repeated clicks on same element)

### Week 6: Custom Events & Goals

**Goal:** Let users track conversions and business metrics.

- [ ] **Goals/conversions UI** — Define and track goals
  - Goal types: pageview (URL match), custom event, duration threshold
  - Goal creation form in Settings
  - Conversion rate display in KPI cards
  - Goal completion timeline chart
  - New API: `GET/POST /api/projects/{id}/goals`
- [ ] **Custom event explorer** — View and filter custom events
  - Event name list with counts
  - Property breakdown per event
  - Revenue tracking (if event has monetary value)
  - New page: `/events`
- [ ] **Funnel builder** — Multi-step conversion analysis
  - Define funnel steps (page visits or custom events)
  - Visualize drop-off between steps
  - Compare funnel performance over time
  - New page: `/funnels`

### Week 7: Data Fetching & Performance

**Goal:** Make the dashboard fast and reliable.

- [ ] **Migrate to SWR or React Query** — Replace raw fetch() calls
  - Automatic caching, revalidation, retry
  - Optimistic updates for settings changes
  - Shared cache across components (avoid duplicate requests)
  - Background refresh without loading states
- [ ] **API response caching** — Server-side caching for expensive queries
  - Cache stats queries for 60 seconds
  - Cache-Control headers for CDN
  - Invalidate on new events (via ClickHouse polling)
- [ ] **Chart performance** — Optimize for large datasets
  - Data point aggregation (don't send 10K points for year view)
  - Virtual scrolling for long tables
  - Lazy load below-the-fold components
- [ ] **Bundle optimization**
  - Enforce <5KB tracker bundle (CI check)
  - Code-split dashboard pages
  - Lazy-load recharts and rrweb-player

### Week 8: Team Collaboration & User Management

**Goal:** Multi-user access for teams.

> [!note] Superseded 2026-06-23 by the auth-brain cutover (PRs #15-#17). Membership, invitations, and roles are now owned centrally by auth-brain (auth.lumitra.co), not by this app. The in-app routes + Settings team UI were removed; Settings links to the account portal. The two items below are therefore NOT built here. Re-homing any member-management UI into this dashboard is blocked on auth-brain shipping list-members/list-invitations endpoints (the SDK currently exposes only verifySession/getCurrentUser/can, no list methods).

- [x] **Invite team members** (moved to auth-brain), Email-based invitation flow, owned by auth-brain machine admin API (`POST /invitations`)
- [x] **Team management UI** (moved to auth-brain), Member list, role changes, removal live in the auth.lumitra.co account portal (`POST/DELETE /memberships`)
- [ ] **Activity log** — Track who did what
  - API key created/revoked events
  - Project settings changed
  - Team member added/removed
- [ ] **Shared dashboard links** — Public/read-only dashboard URLs
  - Toggle public access per project
  - Custom vanity URL
  - Embeddable iframe snippet

## Month 3: Differentiation & Production Hardening (Weeks 9-12)

### Week 9: AI-Powered Insights

**Goal:** Differentiate from competitors with AI analytics.

- [ ] **Automated insights** — AI-generated summaries on dashboard
  - "Traffic is up 34% vs last week, driven by a referral from producthunt.com"
  - "Your /pricing page has 78% bounce rate — significantly higher than average"
  - "Mobile visitors increased 2x since Tuesday"
  - Claude API integration for natural language summaries
  - Scheduled daily/weekly insight generation
- [ ] **Anomaly detection** — Automatic alerts for unusual patterns
  - Traffic spike/drop detection (standard deviation threshold)
  - Email/webhook notifications
  - Visual indicators on timeline chart
- [ ] **AI chat** — Ask questions about your analytics
  - "What's my best traffic source this month?"
  - "Which pages have the highest bounce rate?"
  - Powered by Claude with ClickHouse query generation
  - New page: `/insights` or sidebar panel

### Week 10: Developer Experience & API

**Goal:** Make the platform developer-friendly.

- [ ] **API documentation** — Interactive API docs
  - OpenAPI/Swagger spec auto-generated
  - Try-it-out playground
  - Code examples in JS, Python, curl
  - Published at `/docs` or separate subdomain
- [ ] **Embeddable widgets** — Drop-in analytics widgets
  - `<script>` tag for visitor counter
  - Embeddable chart iframe (public projects only)
  - React/Vue component library for custom integrations
- [ ] **Webhook notifications** — Event-driven integrations
  - Goal completion webhooks
  - Traffic threshold alerts
  - New API: `POST /api/projects/{id}/webhooks`
- [ ] **Server-side tracking** — Backend event collection
  - `POST /api/collect` already works for server-side
  - Add official Node.js SDK wrapper
  - Documentation for Next.js/Express/NestJS integration

### Week 11: Production Hardening

**Goal:** Make it reliable for paying customers.

- [ ] **Database schema migrations** — Automated DDL management
  - Migration files for Postgres schema changes
  - ClickHouse schema versioning
  - Run on deploy (not manual setup.sh)
  - Rollback support
- [ ] **Monitoring & alerting** — Know when things break
  - Health check endpoint (`GET /api/health`)
  - ClickHouse query latency monitoring
  - Disk usage alerts (ClickHouse data retention)
  - Uptime monitoring integration (e.g., Lumitra's own tools)
- [ ] **Backup strategy** — Protect customer data
  - PostgreSQL daily backups (pg_dump → S3/R2)
  - ClickHouse backup (clickhouse-backup tool)
  - Automated backup verification
  - Point-in-time recovery documentation
- [ ] **Data retention policies** — Configurable per project
  - Default: 12 months (already in TTL)
  - User-configurable retention period
  - GDPR data deletion requests
  - Data export before deletion

### Week 12: Onboarding & Polish

**Goal:** First-time user experience that converts.

- [ ] **Onboarding flow** — Guide new users to value
  - Step 1: Create project (name + domain)
  - Step 2: Install tracker (copy snippet, framework-specific guides)
  - Step 3: Verify first event (real-time check with countdown)
  - Step 4: Explore dashboard (guided tour of features)
  - Progress indicator in sidebar
- [ ] **Framework integration guides** — One-page setup docs
  - Next.js, React, Vue, Svelte, Astro, WordPress, Shopify
  - Copy-paste snippets with project-specific values
  - Verification button to test integration
- [ ] **Email reports** — Scheduled analytics summaries
  - Weekly/monthly email with KPI summary
  - Configurable per project
  - Beautiful email template (reuse Email Editor?)
- [ ] **Landing page for analytics.lumitra.co** — Public marketing page
  - Feature showcase, pricing, demo dashboard
  - SEO-optimized for "privacy-first analytics", "open-source analytics"
  - Unauthenticated visitors see marketing page, not login

## Design Principles

Based on competitive analysis (Plausible, Umami, Fathom, PostHog):

1. **Single-page dashboard** — Everything important visible without navigation
2. **Click-to-filter** — Any table row becomes a filter for the whole dashboard
3. **Data density over chrome** — Minimal borders, spacing defines structure
4. **One accent color** — Blue line on dark background, no rainbow charts
5. **Skeleton loading** — Never show "Loading..." text
6. **URL-encoded state** — Every filter/date combination is a shareable URL
7. **<3KB tracker** — Competitive with Plausible (~1KB) and Umami (~2KB)
8. **Privacy by default** — Cookie-free, GDPR-compliant, no consent banner needed

## Technical Architecture Changes

### Frontend
- **Current:** Raw `fetch()` + `useState` → **Target:** SWR/React Query for data fetching
- **Current:** Recharts LineChart only → **Target:** Area charts, bar charts, world map, sparklines
- **Current:** Dark-only → **Target:** Dark + light mode with system preference detection
- **Current:** No URL state → **Target:** All filters encoded in URL search params

### Backend
- **Current:** Stub GeoIP → **Target:** MaxMind GeoLite2 integration
- **Current:** Manual schema setup → **Target:** Automated migrations on deploy
- **Current:** No caching → **Target:** Server-side response caching (60s TTL)
- **Current:** No real-time → **Target:** WebSocket/polling for live counters

### Tracker SDK
- **Current:** No referrer/UTM parsing → **Target:** Full attribution tracking
- **Current:** No server-side SDK → **Target:** Node.js SDK wrapper
- **Current:** No bundle size CI → **Target:** <3KB gzip enforced in CI

### Infrastructure
- **Current:** Manual `./deploy.sh` → **Target:** CI/CD on git push
- **Current:** No backups → **Target:** Automated PG + ClickHouse backups
- **Current:** No monitoring → **Target:** Health checks + alerting

## Success Criteria (End of Q2)

After 3 months, the analytics platform should:

1. **Match Plausible/Umami feature parity** on core analytics
2. **Differentiate with session replay + heatmaps + AI insights** (unique combination)
3. **Support 5+ active projects** with team members
4. **Handle 100K events/day** without performance degradation
5. **Pass a "demo to client" test** — polished enough to sell as a service
6. **Self-service onboarding** — new user can set up tracking in under 5 minutes
